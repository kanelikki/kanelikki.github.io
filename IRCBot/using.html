How to use

Important guide for Linux

Currently, we have dependency on icu package.

Before start, you need to add execution permit to your execution file:

chmod u+x ./IRCBot
./IRCBot

You can use screen or tmux.

For Debian, Ubuntu, Mint

sudo apt-get install icu

If you want to use dependent version:

sudo apt-get install dotnet

For Red Hat, Fedora, CentOS

sudo dnf install icu

If you want to use dependent version:

sudo dnf install dotnet

Important Setting with dependent version! Only for Red Hat Line(RHEL, Fedora, CentOS), Add ~/.bashrc to this line:

EXPORT DOTNET_ROOT="/usr/lib64/dotnet";

Which can be added by this way:

echo 'EXPORT DOTNET_ROOT="/usr/lib64/dotnet";' >> ~/.bashrc

Should I download independent package?

If you'll not use program never, ever on linux, which use .Net Core, you don't need to install. But since C# is major language and .Net Core is fresh but still good, you may need in the future. It depends on you (or "root" user).

Of course if you can't install any package, then remove the package.

Command Usage

./IRCBot [-start] [-encoding Encoding]

All parameters are optional.

-start : Load setting file and start directly without setting UI. It won't work without config file.

-encoding : Set Encoding.

Note

There're no standard about IRC encoding and encoding problem occurs between clients. Check servers' recommendation first. If it won't work, try changing encoding. You can see the available list of encoding by writing IRCbot -encoding (without encoding or wrong encoding)

Setting

Press up/down arrow to select. You can navigate on the channel list with left/right arrow.

Select Connect button and press Enter to save and connect. Cancel button and press Entern to cancel.

Setting File(config.json) can be edited manually, but be careful to edit it right.

Adding/Removing Modules

It's simple, just copy the module to Modules folder.

All modules are dll files. If the module is not proper, it won't be loaded.

NOTE: You can't add/remove modules at runtime. After adding or removing the module, you must restart the bot to apply it.

If you want to make your own module, of course you can do it easily.
